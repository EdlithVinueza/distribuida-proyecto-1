# GUÍA DE COMANDOS DOCKER PARA SUBIR A LA NUBE (DOCKER HUB)

Esta guía te ayudará a construir (build), subir (push) y limpiar (borrar) las imágenes
de tus 3 microservicios principales: app-authors, app-books y app-customers.

---
## PASO 0: INICIAR SESIÓN EN DOCKER HUB
Antes de poder subir imágenes, debes iniciar sesión en tu terminal.
Comando:
docker login
(Te pedirá tu usuario de Docker Hub, ej: eavz777, y tu contraseña)

ADVERTENCIA DE SEGURIDAD (Si usas una PC ajena o del laboratorio):
Cuando termines de trabajar, SIEMPRE cierra tu sesión para que nadie más 
pueda modificar tus contenedores en la nube. Para salir, ejecuta:
docker logout


---
## SERVICIO 1: APP-AUTHORS

1. Construir la imagen con la etiqueta de tu cuenta:
(Ejecutar desde la raíz del proyecto distribuda-proyecto-1)
docker build -f app-authors/src/main/docker/Dockerfile.jvm -t eavz777/app-authors:latest ./app-authors

2. Subir (Cargar) la imagen a Docker Hub:
docker push eavz777/app-authors:latest

3. Borrar la imagen localmente (para liberar espacio o forzar a descargarla de nuevo):
docker rmi eavz777/app-authors:latest

## COMANDOS ÚTILES EXTRAS

- Ver todas las imágenes que tienes actualmente en tu computadora:
  docker images

- Ver qué contenedores están corriendo actualmente:
  docker ps

- Forzar el borrado de una imagen si te dice que está en uso (-f de force):
  docker rmi -f edlith777/app-authors:latest

- Limpiar tu sistema (Borra contenedores detenidos, redes sin uso e imágenes colgantes):
  docker system prune
